# Demo UI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a modern, bilingual, dark/light single-page demo dashboard (`demo-ui/`) that shows the Twilio Proxy workflow live, manages the Sync `lookup` mappings, and stages Courier/Buyer phones.

**Architecture:** A self-contained Next.js (App Router) app at repo root. Client-rendered pages give the SPA feel; server Route Handlers hold Twilio credentials for mapping CRUD and for minting a short-lived Sync access token. The browser subscribes directly to a Twilio Sync **Stream** (`demo-events`) for the live feed. The existing Twilio Functions are extended (additively, best-effort) to publish events to that Stream.

**Tech Stack:** Next.js 15 (App Router, TypeScript), Tailwind CSS v4, shadcn/ui (Radix), i18next + react-i18next, twilio (server SDK), twilio-sync (browser SDK), Vitest + React Testing Library. Backend: existing Twilio Serverless Functions (twilio-run, Jest).

## Global Constraints

- **No real phone numbers in any git-tracked file.** Only fictional `+1555…` placeholders. Real numbers live only in Twilio Sync (live data) / in transit / gitignored env files.
- **Never commit secrets.** `demo-ui/.env.local` is gitignored; verify no `.env*` file is staged before every commit.
- **Commit only when steps say to** (this plan's commit steps are the exception to the "ask first" rule — the user has approved plan execution).
- Commit messages end with: `Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>`.
- **Publishing live-feed events from Twilio Functions must be best-effort:** a Sync publish failure must never throw or change the Function's response.
- **Dark theme is the default.** SPA feel: client-side routing, no full page reloads.
- Node ^20 || ^22 (repo `.nvmrc` / engines). demo-ui uses the same.
- Brand colors are exact: Twilio Red `#EF223A`, Ink `#000D25`, White `#FFFFFF`.
- Two languages: English (`en`) and Simplified Chinese (`zh`). Every visible string is a translation key present in both locale files.

```ts
// ---------------------------------------------------------------------------
// Live-feed events (published to a Sync Stream, consumed by the demo-ui).
// ---------------------------------------------------------------------------

export type DemoEventType =
  | 'oos.prompt'
  | 'lookup.request'
  | 'lookup.result'
  | 'resolution.stored'
  | 'oos.autocreate';

export type DemoEvent = {
  type: DemoEventType;
  ts: string;
  callSid?: string;
  from?: string;
  to?: string;
  digits?: string;
  realNumber?: string;
  note?: string;
};

// Name of the Sync Stream the demo-ui subscribes to. Keep in sync with the
// demo-ui token scope (demo-ui/app/api/token/route.ts).
export const EVENTS_STREAM_NAME = 'demo-events';

/**
 * Publish a demo event to the `demo-events` Sync Stream. BEST-EFFORT: this must
 * never throw and never change a webhook's behavior. If the stream doesn't
 * exist yet the first publish 404s, so we create it and retry once; any final
 * error is swallowed.
 */
export async function publishEvent(
  client: SyncClient,
  syncServiceSid: string,
  event: DemoEvent
): Promise<void> {
  const service = () => client.sync.v1.services(syncServiceSid);
  try {
    await service()
      .syncStreams(EVENTS_STREAM_NAME)
      .streamMessages.create({ data: event as unknown as Record<string, unknown> });
    return;
  } catch {
    // Stream likely missing — create it below and retry once.
  }
  try {
    await service().syncStreams.create({ uniqueName: EVENTS_STREAM_NAME });
  } catch {
    // already exists / transient — ignore
  }
  try {
    await service()
      .syncStreams(EVENTS_STREAM_NAME)
      .streamMessages.create({ data: event as unknown as Record<string, unknown> });
  } catch {
    // Give up silently: publishing must never break the call flow.
  }
}
```

```tsx
import { useTheme } from 'next-themes';
import { useEffect, useState } from 'react';
import { Button } from '@/components/ui/button';

export function ThemeSwitch() {
  const { theme, setTheme } = useTheme();
  const [mounted, setMounted] = useState(false);
  useEffect(() => setMounted(true), []);

  const isDark = theme === 'dark';
  return (
    <Button
      variant="outline"
      size="sm"
      aria-label="Toggle theme"
      onClick={() => setTheme(isDark ? 'light' : 'dark')}
    >
      {mounted ? (isDark ? '🌙 Dark' : '☀️ Light') : '🌙 Dark'}
    </Button>
  );
}
```

```ts
export type DemoEventType =
  | 'oos.prompt'
  | 'lookup.request'
  | 'lookup.result'
  | 'resolution.stored'
  | 'oos.autocreate';

export const EVENT_TYPES: DemoEventType[] = [
  'oos.prompt',
  'lookup.request',
  'lookup.result',
  'resolution.stored',
  'oos.autocreate',
];

export type DemoEvent = {
  type: DemoEventType;
  ts: string;
  callSid?: string;
  from?: string;
  to?: string;
  digits?: string;
  realNumber?: string;
  note?: string;
};

export function parseEvent(raw: unknown): DemoEvent | null {
  if (!raw || typeof raw !== 'object') return null;
  const e = raw as Record<string, unknown>;
  if (typeof e.type !== 'string') return null;
  if (!EVENT_TYPES.includes(e.type as DemoEventType)) return null;
  if (typeof e.ts !== 'string') return null;
  return e as DemoEvent;
}
```

```ts
// Mask the middle of a phone number for display. Keeps a leading prefix and the
// last two digits: "+15551112222" -> "+1555•••••22". Short values (<= 5 chars)
// are returned as-is; empty values render as an em dash.
export function maskNumber(n?: string): string {
  if (!n) return '—';
  if (n.length <= 5) return n;
  const prefix = n.slice(0, 5);
  const last2 = n.slice(-2);
  const hidden = '•'.repeat(Math.max(0, n.length - 7));
  return `${prefix}${hidden}${last2}`;
}
```

```ts
export async function GET() {
  const {
    TWILIO_ACCOUNT_SID,
    TWILIO_API_KEY,
    TWILIO_API_SECRET,
    TWILIO_SYNC_SERVICE_SID,

    !TWILIO_SYNC_SERVICE_SID

    return Response.json(
      { error: 'Missing Twilio env configuration' },
      { status: 500 }
    );
  }

  const AccessToken = twilio.jwt.AccessToken;
  const SyncGrant = AccessToken.SyncGrant;
  const identity = 'demo-ui-viewer';

  const token = new AccessToken(
    TWILIO_ACCOUNT_SID,
    TWILIO_API_KEY,
    TWILIO_API_SECRET,
    { identity, ttl: 3600 }
  );
  token.addGrant(new SyncGrant({ serviceSid: TWILIO_SYNC_SERVICE_SID }));

  // Referenced so the scope is explicit for readers/tooling.
  void EVENTS_STREAM_NAME;

  return Response.json({ token: token.toJwt(), identity });
```

```ts
    p.length === 2 &&

    .services(syncServiceSid())

    .syncMapItems.list({ limit: 200 });
  return Response.json({
```

```ts
    Array.isArray(p) &&
    p.length === 2 &&
    p.every((x) => typeof x === 'string' && x.length > 0)

export async function PUT(
  req: Request,
  { params }: { params: Promise<{ code: string }> }
) {
```

```tsx
const LABEL_KEY: Record<DemoEventType, string> = {

function isProminent(t: DemoEventType) {

}

export function LiveFeedView({
  events,

  events: DemoEvent[];
  status: 'connecting' | 'open' | 'error';
}) {
  const { t } = useTranslation();
  return (

        <h2 className="text-lg font-bold">{t('liveFeed.title')}</h2>

      {events.length === 0 ? (
        <p className="opacity-60">{t('liveFeed.empty')}</p>
      ) : (
        <ul className="space-y-2">
          {events.map((e, i) => {
            const prominent = isProminent(e.type);
            const detail =
```

```ts
'use client';

import { useEffect, useRef, useState } from 'react';
import { SyncClient } from 'twilio-sync';
import { parseEvent, type DemoEvent } from '@/lib/events';

const STREAM_NAME = 'demo-events'; // mirror backend EVENTS_STREAM_NAME

export function useSyncStream() {
  const [events, setEvents] = useState<DemoEvent[]>([]);
  const [status, setStatus] = useState<'connecting' | 'open' | 'error'>(
    'connecting'
  );
  const clientRef = useRef<SyncClient | null>(null);

  useEffect(() => {
    let cancelled = false;
    (async () => {
      try {
        const res = await fetch('/api/token');
        const { token } = await res.json();
        if (cancelled) return;
        const client = new SyncClient(token);
        clientRef.current = client;
        const stream = await client.stream(STREAM_NAME);
        if (cancelled) return;
        setStatus('open');
        stream.on('messagePublished', (args: { message: { data: unknown } }) => {
          const parsed = parseEvent(args.message.data);
          if (parsed) setEvents((prev) => [parsed, ...prev].slice(0, 100));
        });
      } catch {
        if (!cancelled) setStatus('error');
      }
    })();
    return () => {
      cancelled = true;
      clientRef.current?.shutdown?.();
    };
  }, []);

  return { events, status };
}
```

```ts
export type CardStyle = 'parcel' | 'boarding' | 'plain';

export type Settings = {
  cardStyle: CardStyle;
  courierLabel: string;
  buyerLabel: string;
};

const KEY = 'demo-ui-settings';

export const DEFAULT_SETTINGS: Settings = {
  cardStyle: 'parcel',
  courierLabel: 'Courier',
  buyerLabel: 'Buyer',
};

export function loadSettings(): Settings {
  if (typeof window === 'undefined') return DEFAULT_SETTINGS;
  try {
    const raw = window.localStorage.getItem(KEY);
    if (!raw) return DEFAULT_SETTINGS;
    const parsed = JSON.parse(raw) as Partial<Settings>;
    return { ...DEFAULT_SETTINGS, ...parsed };
  } catch {
    return DEFAULT_SETTINGS;
  }
}

export function saveSettings(s: Settings): void {
  if (typeof window === 'undefined') return;
  try {
    window.localStorage.setItem(KEY, JSON.stringify(s));
  } catch {
    /* ignore */
  }
}
```

```ts
'use client';

import { useCallback, useEffect, useState } from 'react';
import {
  DEFAULT_SETTINGS, loadSettings, saveSettings, type Settings,
} from '@/lib/settings';

export function useSettings() {
  const [settings, setSettings] = useState<Settings>(DEFAULT_SETTINGS);

  useEffect(() => setSettings(loadSettings()), []);

  const update = useCallback((patch: Partial<Settings>) => {
    setSettings((prev) => {
      const next = { ...prev, ...patch };
      saveSettings(next);
      return next;
    });
  }, []);

  return { settings, update };
}
```

````markdown
# demo-ui

A Next.js dashboard for the Twilio Proxy dynamic-session workflow: live call-flow
feed (via a Sync Stream), lookup-mapping CRUD, reserved-number cards, and a
Courier/Buyer phone staging area. Bilingual (EN / 中文), dark/light themes.

## Setup

```bash
cd demo-ui
cp .env.local.example .env.local   # fill in Twilio account, Sync SID, API key/secret
npm install
npm run dev                        # http://localhost:3000
```

| Var | Purpose |
|-----|---------|
| `TWILIO_ACCOUNT_SID` / `TWILIO_AUTH_TOKEN` | server-side mapping CRUD |
| `TWILIO_SYNC_SERVICE_SID` | the Sync service holding the `lookup` Map + `demo-events` Stream |
| `TWILIO_API_KEY` / `TWILIO_API_SECRET` | mint browser Sync access tokens |

```
````
